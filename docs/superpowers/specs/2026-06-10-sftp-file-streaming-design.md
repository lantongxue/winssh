# SFTP File Streaming Design

**Date:** 2026-06-10

## Goal

Reduce UI stalls and memory spikes when opening and saving remote text files in the SFTP file
editor.

The editor remains a text-only Monaco editor for complete remote files. This design does not turn
the app into a large-file or binary editor.

## Current Behavior

- `SessionManager.readFile()` reads the remote file in 32 KB SFTP chunks, concatenates all buffers,
  decodes the complete buffer, and returns one `{ content, encoding }` object over IPC.
- `WorkbenchSftpFileMonacoEditor` waits for the complete Promise result, then calls
  `model.setValue(content)`.
- `SessionManager.writeFile()` receives the complete text string over IPC, encodes the full string
  into one buffer, then writes it to SFTP in 32 KB chunks.
- Progress UI is driven through the existing `sftp:transfer` event with `localPath: '__editor__'`.

The remote SFTP layer is already chunked, but the main-to-renderer and renderer-to-main file editor
contracts are still whole-file contracts.

## Selected Approach

Replace the public whole-file editor API with an editor-specific streaming API.

Old API contracts are removed from shared/preload/renderer and main IPC:

- Remove `sftp.readFile(sessionId, remotePath)`.
- Remove `sftp.writeFile(sessionId, remotePath, contents, encoding)`.
- Remove `sftp:readFile`.
- Remove `sftp:writeFile`.

Low-level helper functions may still read or write SFTP handles in chunks, but they must not expose a
whole-file string Promise as the file editor API.

## Stream API

Add shared types for editor file streams:

- `SftpFileStreamId`: unique request id generated by the main process.
- `SftpFileReadStreamStart`: `{ streamId, sessionId, remotePath, fileName, total, encoding }`.
- `SftpFileChunkEvent`: `{ streamId, sessionId, remotePath, chunk, transferred, total }`.
- `SftpFileStreamStateEvent`:
  `{ streamId, sessionId, remotePath, direction, status, transferred, total, error? }`.
- `SftpFileWriteStreamStart`: `{ streamId, sessionId, remotePath }`.

Add `WinsshApi['sftp']` methods:

- `openFileReadStream(sessionId, remotePath): Promise<SftpFileReadStreamStart>`
- `openFileWriteStream(sessionId, remotePath, encoding): Promise<SftpFileWriteStreamStart>`
- `writeFileChunk(streamId, chunk): Promise<void>`
- `closeFileWriteStream(streamId): Promise<void>`
- `cancelFileStream(streamId): void`
- `onFileChunk(callback): Unsubscribe`
- `onFileStreamState(callback): Unsubscribe`

The preload bridge maps these methods to new IPC channels and event subscriptions. Renderer code
continues to access the bridge only through `features/sftp/api/sftp-client`.

## Read Flow

1. The renderer calls `openFileReadStream(sessionId, remotePath)`.
2. The main process validates the session and path, stats the remote file, rejects directories, and
   creates a read stream task keyed by `streamId`.
3. The main process opens the SFTP handle and reads fixed-size byte chunks.
4. The main process decodes incrementally and emits text chunk events for the matching `streamId`.
5. The renderer buffers incoming chunks briefly and appends them to the Monaco model in animation
   frame batches.
6. The main process emits a completed state event when the handle is closed successfully.
7. The renderer marks the editor clean only after completion and stores the completed text snapshot
   as `savedContent`.

Incremental decoding must preserve multi-byte character boundaries across SFTP byte chunks. Add a
small encoding helper that can inspect an initial byte sample, choose the same encoding family as the
current `smartDecode()` path, and then feed subsequent byte chunks through one decoder instance. UTF-8
and UTF-16 BOM variants keep their existing behavior. Non-UTF encodings continue to use `iconv-lite`
when detected from the initial sample. Unknown or unsupported encoding still falls back to UTF-8,
matching current behavior.

## Save Flow

1. The renderer calls `openFileWriteStream(sessionId, remotePath, encoding)`.
2. The main process creates a write stream task and opens the remote SFTP file in write mode.
3. The renderer reads the current Monaco text and sends it in bounded chunks through
   `writeFileChunk(streamId, chunk)`.
4. The main process encodes and writes each chunk before resolving that chunk call.
5. The renderer calls `closeFileWriteStream(streamId)` after all chunks are acknowledged.
6. The main process closes the SFTP handle and emits a completed state event.
7. The renderer marks the editor clean only after the close completes.

Chunk acknowledgement provides natural backpressure: the renderer does not enqueue the entire file
into IPC at once.

## Cancellation And Errors

- Cancellation uses `streamId`, not `sessionId:remotePath`, so a stale close action cannot cancel a
  newer open of the same path.
- Closing an editor while a read is active calls `cancelFileStream(streamId)`.
- Cancelling a write closes the SFTP handle and leaves the editor dirty.
- Main process errors emit `status: 'error'` with an error message and reject the initiating or
  current chunk call.
- Renderer read errors show the existing load failure state and do not populate Monaco with partial
  content as a saved file.
- Renderer save errors keep the dirty marker and show the existing save failure toast.

## Progress

Keep the existing editor progress UI and transfer panel behavior by bridging stream state into
`sftp:transfer` events with `localPath: '__editor__'`.

The new stream state event is the precise lifecycle contract for the editor. The old transfer event
remains a UI compatibility event for progress surfaces.

## Renderer Editor Behavior

- Replace React Query `useQuery` for file contents with explicit stream lifecycle state.
- Keep React Query keys for cache invalidation only where useful, but do not cache complete file
  contents through the old `sftpFile` query result.
- Use Monaco batch appends instead of one large `setValue`.
- Ignore chunks whose `streamId` does not match the active editor load.
- Keep the editor read-only or loading-covered until the read stream completes, so users cannot edit
  an incomplete model.
- Preserve current theme, font, temporary zoom, keyboard save, cancel button, and text-only language
  detection behavior.

## Main Process Behavior

- Add a stream task map in `SessionManager` for editor reads and writes.
- Each task owns its SFTP handle, `AbortController`, transferred byte count, and stream metadata.
- The session application service delegates stream methods to `SessionManager`.
- `register-session-ipc.ts` exposes only the new stream handlers for file editing.
- Stream cleanup always closes SFTP handles and removes task state.

## Tests

Main process tests:

- Encoding helper tests cover UTF-8 multi-byte characters split across chunks, UTF-16 BOM detection,
  and a non-UTF sample detected through the existing charset fallback.
- Starting a read stream emits chunks before completion and does not return a whole file string.
- Cancelling by `streamId` stops the read and cleans the task.
- Starting a write stream writes acknowledged chunks incrementally and closes the handle on complete.
- Write cancellation closes the handle and leaves the task cleaned up.

Renderer tests:

- The SFTP Monaco editor appends streamed chunks into the model and marks clean after completion.
- The editor ignores chunks from stale `streamId` values.
- Saving sends bounded chunks and keeps the editor dirty until close succeeds.
- Save failure keeps dirty state and shows the existing failure toast.

Shared/preload tests:

- `WinsshApi['sftp']` and the mock helper no longer expose `readFile` or `writeFile`.
- New stream event subscriptions unsubscribe correctly.

## Out Of Scope

- Binary SFTP editing.
- Virtualized large-file editing.
- Database schema changes.
- SFTP explorer upload/download behavior.
- Port forwarding, terminal, and session lifecycle changes outside file editor streaming.
