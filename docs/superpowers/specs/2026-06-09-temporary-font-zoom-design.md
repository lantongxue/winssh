# 临时字体缩放设计

**日期:** 2026-06-09

## 目标

在终端页面和 SFTP 文件编辑器页面支持按住 Ctrl 或 Cmd 并滚动鼠标滚轮来临时放大或缩小字体。

## 范围

- SSH 终端页和本地终端页通过共享的 `TerminalSurface` 支持缩放。
- SFTP 远程文件编辑器通过 Monaco 编辑器实例支持缩放。
- 缩放只影响当前打开的页面实例，不写入设置、不更新数据库。
- 关闭页面或重启应用后，字号恢复为设置中的 `terminalFontSize`。
- 字号沿用现有合法范围：最小 10，最大 24，每次滚轮步进 1。

## 行为

- 用户在终端或编辑器内容区域按住 Ctrl/Cmd 并滚动时，拦截默认行为。
- 向上滚动增大字号，向下滚动减小字号。
- 只影响当前页面：一个终端页缩放不会影响其他终端页或编辑器页。
- 基础字号仍来自当前设置和主题解析结果；临时缩放以偏移量叠加在基础字号上。
- 设置中的字号变化后，页面继续保留当前临时偏移，并重新夹紧到 10-24 范围。

## 架构

新增一个小型 renderer 工具模块，集中定义字体缩放范围、滚轮方向解析、字号夹紧和偏移计算。`TerminalSurface` 维护当前终端页面的临时偏移，并把覆盖字号传给 `useTerminal`；Monaco 文件编辑器维护自己的临时偏移，并把计算后的字号传给 `editor.updateOptions()`。

## 测试

- 为字体缩放工具模块添加单元测试，覆盖方向、夹紧和偏移计算。
- 为 `TerminalSurface` 添加组件测试，验证 Ctrl+滚轮只调整当前终端实例的 `fontSize`，并且不调用 settings 更新。
- 为 SFTP Monaco 文件编辑器添加组件测试，验证 Ctrl+滚轮只调整当前编辑器实例的 `fontSize`，并且不调用 settings 更新。

## 非目标

- 不增加新的设置项。
- 不持久化用户的滚轮缩放结果。
- 不改变编辑器字体族、主题、行高或终端 keep-mounted 策略。
